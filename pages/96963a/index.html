使用PG_RMAN对PostgreSQL进行全量备份

# 使用PG_RMAN对PostgreSQL进行全量备份

# pg_rman特点

  • 使用简单，一个命令即可完成备份和恢复。
  • 支持在线全备，增量备份，归档备份。
  • 支持备份压缩，通过gzip工具实现页内压缩。
  • 自动备份维护，自动删除过期的WAL备份文件。
  • 支持备份验证。
  • 恢复期间无事务丢失，支持基于PITR的配置文件生成器。
  • 需要挂载备份介质到本机

# 初始化备份目录

mkdir -p /backup/pgsql5432 
/usr/pgsql-9.5/bin/pg_rman init -D /pgsql/pgsql5432 -B /backup/pgsql5432

# 全备实例

export PGPASSWORD=postgres         # 超级用户密码  

pg_rman backup \    
-b full \                          # 全量备份   
-B /backup/pgsql5432 \       # 备份目录  
-D /pgsql/pgsql5432 \   # 备库的$PGDATA  
-s \                               # 备份pg_log  
-Z \                               # 压缩  
--keep-data-generations=3 \        # 保留3个全量备份，删除不需要的全量备份  
--keep-data-days=10 \              # 保证能恢复到10天内的任意时间点，删除不需要的  
--keep-arclog-files=30 \           # 保留最近30个归档文件  
--keep-arclog-days=20 \            # 保留20天内的归档文件  
--keep-srvlog-files=10 \           # 保留最近10个日志文件(pg_log)  
--keep-srvlog-days=20 \            # 保留20天内的日志文件(pg_log)  
--standby-host=127.0.0.1 \         # 如何连接standby  
--standby-port=5432 \              # 如何连接standby  
-h 127.0.0.1 \                     # 如何连接primary  
-p 1921 \                          # 如何连接primary  
-U postgres \                      # 如何连接primary, standby(超级用户)  
-d postgres                        # 如何连接primary, standby(database name)

# 备份集校验

每次备份完，必须要做一次校验，否则备份集不可用用来恢复，增量备份时也不会用它来做增量比较。

pg_rman validate

# 删除备份集

  1. 按指定时间从catalog删除备份集

    pg_rman delete "2017-08-30 17:27:49"

  2. 根据备份策略来删除备份集

    KEEP_DATA_GENERATIONS = 3 -- 备份集冗余度是3

    KEEP_DATA_DAYS = 10 -- 备份集保留日期是10d
