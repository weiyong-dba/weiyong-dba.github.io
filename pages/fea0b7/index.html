Redis高可用架构之Sentinel配置管理

# Redis高可用架构之Sentinel配置管理

# 结构

sentinel可以让redis实现主从复制，当一个集群中的master失效之后，sentinel可以选举出一个新的master用于自动接替master的工作，集群中的其他redis服务器自动指向新的master同步数据。一般建议sentinel采取奇数台，防止某一台sentinel无法连接到master导致误切换。其结构如下:

# 部署

编译sentinel

  • unzip redis-2.8.zip
  • mv redis-2.8 /usr/local
  • cd /usr/local
  • mv redis-2.8 sentinel
  • cd sentinel
  • make

编辑sentinel.conf

cat <<EOF>sentinel.conf
port 26379
bind 0.0.0.0
dir /tmp
daemonize yes
sentinel down-after-milliseconds identification_master 30000
sentinel parallel-syncs identification_master 1
sentinel failover-timeout identification_master 180000
logfile "/usr/local/sentinel/sentinel.log"
EOF

[Unit]

[Service]
Type=forking
User=redis
Group=redis
ExecStart=/usr/local/sentinel/src/redis-sentinel /usr/local/sentinel/sentinel.conf
PrivateTmp=true
TimeoutSec=60
[Install]
WantedBy=multi-user.target
EOF

添加redis监控

执行sentinel monitor <masterName> <ip> <port> <quorum>，添加redis实例到sentinel。

  • masterName - 监控服务名
  • ip - master实例ip
  • port - master实例端口
  • quorum - 需要投票的sentinel实例数

# 管理

# Sentinel状态持久化

snetinel的状态会被持久化地写入sentinel的配置文件中。每次当收到一个新的配置时，或者新创建一个配置时，配置会被持久化到硬盘中，并带上配置的版本戳。因此可以安全的停止和重启sentinel进程。

# Sentinel作用

  • Master状态检测
  • 如果Master异常，则会进行Master-Slave切换，将其中一个Slave作为Master，将之前的Master作为Slave。
  • Master-Slave切换后，master_redis.conf、slave_redis.conf和sentinel.conf的内容都会发生改变，即master_redis.conf中会多一行slaveof的配置，sentinel.conf的监控目标会随之调换。

# Sentinel命令

登录到sentinel节点后执行的命令，比如执行redis-cli -h 192.168.10.203 -p 26379

* PING - 返回PONG
* SENTINEL masters - 列出所有被监视的主服务器，以及这些主服务器的当前状态。
* SENTINEL slaves <master name> - 列出给定主服务器的所有从服务器的当前状态。
* SENTINEL get-master-addr-by-name <master name> - 返回给定名字的主服务器的IP地址和端口号。
* SENTINEL reset <pattern> - 重置所有名字和给定模式pattern相匹配的主服务器。 pattern参数是一个Glob风格的模式。重置操作清楚主服务器目前的所有状态，包括正在执行中的故障转移，并移除目前已经发现和关联的，主服务器的所有从服务器和Sentinel。
* SENTINEL failover <master name> - 当主服务器失效时，在不询问其他 Sentinel意见的情况下，强制开始一次自动故障迁移。（不过发起故障转移的Sentinel 会向其他Sentinel发送一个新的配置，其他Sentinel会根据这个配置进行相应的更新，slave-priority低的slave优先提升为master）。
* SENTINEL MONITOR <name> <ip> <port> <quorum> - 这个命令告诉sentinel去监听一个新的master。
* SENTINEL REMOVE <name> - 命令sentinel放弃对某个master的监听。
* SENTINEL SET <name> <option> <value> : 这个命令很像Redis的CONFIG SET命令，用来改变指定master的配置。支持多个optionvalue。
	* 例如以下实例：SENTINEL SET objects-cache-master down-after-milliseconds 1000
只要是配置文件中存在的配置项，都可以用SENTINEL SET命令来设置。这个还可以用来设置master的属性，比如说quorum(票数)，而不需要先删除master，再重新添加master。
	* 例如：SENTINEL SET objects-cache-master quorum 5
* SENTINEL get-master-addr-by-name <master name> - 获取当前的主服务器IP地址和端口号。
* SENTINEL slaves <master name> - 获取所有的Slaves信息。
