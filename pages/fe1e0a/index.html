主从复制搭建及错误处理

# 主从复制搭建及错误处理

# 前言

很多时候生产库一开始没有搭建slave，运行一段时间后感觉不稳妥，晚上开始有点睡不着了，一大早来上班兴冲冲开始搭建slave，本文将带你get这项让你夜晚可以安睡的技能，详细介绍如何为生成库增加slave，开搞。

# 工具准备

  • innobackupex

# 备份Master

nohup innobackupex –defaults-file=/etc/my.cnf –lock-wait-timeout=300 –slave-info –safe-slave-backup –parallel=24 –user=backup –password=backuppass –parallel=24 /tmp/2019-09-12_15-00-00 &
1

备份结束后，将整个备份目录2019-09-12_15-00-00复制到slave。

# 恢复slave

cd /tmp/2019-09-12_15-00-00
innobackupex –apply-log –parallel=24 .
1
2

在slave机器上安装相同版本的mysql，假设数据目录是/mysql/mysql3306。

mv 2019-09-12_15-00-00 /mysql/mysql3306

启动mysql server

# 配置slave

在数据目录中找到xtrabackup_binlog_info文件，该文件记录了binlog的位置。 如：

mysql-bin.000001 40488

登录slave，执行change master to命令。

CHANGE MASTER TO
  MASTER_HOST='master ip',
  MASTER_USER='repl',
  MASTER_PASSWORD='repl',
  MASTER_PORT=3306,
  MASTER_LOG_FILE='mysql-bin.000001',
  MASTER_LOG_POS=40488;
1
2
3
4
5
6
7

启动slave

start slave;

检查是否成功

show slave status\G;

IO、SQL状态都是Yes，即为成功。

  • Slave_IO_Running: Yes
  • Slave_SQL_Running: Yes

# 移除slave

要移除slave，执行reset slave all即可。

# 跳过错误

# 5.6版本解决方法

方法一：忽略错误后，继续同步

该方法适用于主从库数据相差不大，或者要求数据可以不完全统一的情况，数据要求不严格的情况。

stop slave;
set global sql_slave_skip_counter=1;#表示跳过一步错误，后面的数字可变
start slave;
之后再用show slave status\G;查看
1
2
3
4

方式二：重新做主从，完全同步

该方法适用于主从数据库相差较大，或者要求数据完全统一的情况

# 5.7版本解决方法

解决方法一：跳过错误

1、停止slave进程

stop slave;

2、设置事务号，事务号从Retrieved_Gtid_Set获取,在session里设置gtid_next，即跳过这个GTID

set @@SESSION.GTID_NEXT= '85ff3269-b54d-11e7-97a8-fa163e66866b:5';

3、设置空事务

begin;

commit;

4、恢复事务号

set session gtid_next=automatic;

5、启动slave进程

start slave;

解决方法二：重置master方法跳过错误

stop slave;
reset master;
set  @@glocal.gtid_purged='5ff3269-b54d-11e7-97a8-fa163e66866b:5';
start slave;
1
2
3
4

解决方案三：使用pt-slave-restart工具

pt-slave-restart工具的作用是监视某些特定的复制错误，然后忽略，然后再次启动slave进程,例如：忽略所有1062错误，并再次启动slave进程

pt-slave-restart  -S  /mysql/mysqld.sock  --error-numbers=1062  --user=root  --password=
1

# 后记

本文介绍了如何基于master的备份创建slave的方法，基于slave的备份创建slave的方法也是一样的，只是binlog位置需要从文件xtrabackup_slave_info获取而不是xtrabackup_binlog_info，只有在备份的时候增加slave-info选项才会产生xtrabackup_slave_info文件。
