Aerospike管理手册

# Aerospike管理手册

# 查找所有的namespace

  • show namespaces
  • show sets
  • show bins

# 索引管理

创建索引CREATE INDEX <index> ON <ns>[.<set>] (<bin>) <type>

* <index> - 索引名称，在一个命名空间上索引名称必须唯一；
* <ns> - 命名空间；
* <set>设置是可选的，如果没有设置，则索引的记录不属于一个set；
* <bin>根据bin建立索引
* <type>索引存储的类型，必须是：NUMERIC or STRING；NUMERIC只索引bin中类型是NUMERIC的行

# 查询和扫描管理

  • 启用查询跟踪asinfo -v "set-config:context=service;query-job-tracking=true"
  • 停止查询跟踪asinfo -v "set-config:context=service;query-job-tracking=false"
  • 正在运行的查询show queriess
  • 正在运行的扫描show scan
  • 结束正在运行的任务
KILL_QUERY <transaction_id>
KILL_SCAN <scan_id>

# 查询记录

SELECT * FROM <ns>[.<set>]
SELECT <bin>[, <bin>[, ...]] FROM <ns>[.<set>]
SELECT <bin>[, <bin>[, ...]] FROM <ns>[.<set>] WHERE [<predicate>]
<predicate>只能用在加索引的bin上，没有索引或者索引类型不一致的where报错：Error: (201) AEROSPIKE_ERR_INDEX_NOT_FOUND
<bin> BETWEEN <lower> AND <upper>，只有NUMERIC支持范围查询
<bin> = <value>

# 操作记录

  • 插入记录INSERT INTO test.demo (PK, foo, bar) VALUES ('key1', 123, 'abc')
  • 删除记录DELETE FROM <ns>[.<set>] WHERE PK=<key>
  • 删除setasinfo -v "set-config:context=namespace;id=ns1;set=testset;set-delete=true;"

注意：社区版本中删除的记录并不会从SSD中物理删除，需要把SSD数据清空后再加入集群才能真正的删除。但是配置了TTL的会自动的物理删除，再次启动数据会删除。

# 设置

所有set的值和属性只能保持在当前的session里面

aql> get <setting>
aql> get output
OUTPUT = TABLE
aql> set <setting> <value>
aql> set output json
可用的设置：
VIEW
输出模型.
VIEW ( TABLE | JSON )

VERBOSE

# 设置TTL

编辑ttl.lua脚本

function modify_zero_ttl(rec, to_ttl)
        local currTTL = record.ttl(rec)
        if ( currTTL == 0 ) then
                record.set_ttl(rec, to_ttl)
                local result = aerospike:update(rec)
                if ( result ~= nil and result ~= 0 ) then
                        warn("expireRecord:Failed to UPDATE the record: resultCode (%s)", tostring(result))
                end
        end
end
